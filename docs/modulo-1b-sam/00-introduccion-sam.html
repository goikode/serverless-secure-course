📚 ¿Qué es Infrastructure as Code (IaC)?

Infrastructure as Code es la práctica de definir y gestionar infraestructura mediante código versionable, en lugar de procesos manuales.

Ejemplo Visual

❌ Sin IaC (Manual):

Ingeniero → Consola AWS → Click, Click, Click → Infraestructura

✅ Con IaC (Automatizado):

Código (template.yaml) → `sam deploy` → Infraestructura

🤔 ¿Por Qué Usar IaC?

Problema Manual Solución con IaC
"Works on my machine" Mismo código = Mismos recursos siempre
Pasos olvidados Todo está documentado en el template
Cambios sin historial Version control con Git
Deshacer cambios es difícil Rollback automático
Recrear entorno tarda horas sam deploy en minutos
Documentación desactualizada El código ES la documentación

⚡ ¿Qué es AWS SAM?

AWS Serverless Application Model (SAM) es un framework open-source para construir aplicaciones serverless en AWS.

Características Principales

  • 🚀 Sintaxis simplificada - Menos código que CloudFormation
  • 📦 Built-in best practices - Seguridad y permisos por defecto
  • 🧪 Local testing - Ejecuta Lambda localmente sin desplegar
  • 🔍 Debugging - Depura funciones Lambda en tu IDE
  • 📊 Integración CloudFormation - Compatible con todos los recursos AWS
  • 🎯 CLI poderosa - sam build, sam deploy, sam logs

🏗️ Anatomía de un Template SAM

Un template SAM tiene 6 secciones principales:

AWSTemplateFormatVersion: '2010-09-09'
Transform: AWS::Serverless-2016-10-31
Description: Descripción de tu aplicación

Parameters:    # Valores configurables (como variables)
  StudentPrefix:
    Type: String

Globals:       # Configuración compartida para todos los recursos
  Function:
    Timeout: 10
    Runtime: nodejs20.x

Resources:     # Los recursos AWS que quieres crear
  MyLambda:
    Type: AWS::Serverless::Function
    Properties:
      # ...

Outputs:       # Valores que se muestran después del deploy
  ApiUrl:
    Value: !Sub 'https://${MyApi}.execute-api...'

🆚 SAM vs CloudFormation vs Terraform

Framework Lenguaje Serverless Focus AWS-Specific Learning Curve
SAM YAML ✅✅✅ ✅ AWS only 🟢 Baja
CloudFormation YAML/JSON 🟡 ✅ AWS only 🟡 Media
Terraform HCL 🟡 ❌ Multi-cloud 🔴 Alta
Serverless Framework YAML ✅✅✅ ❌ Multi-cloud 🟡 Media

💡 Para este curso usamos SAM porque:
- ✅ Diseñado específicamente para serverless AWS
- ✅ Sintaxis simple y clara
- ✅ Integración nativa con AWS
- ✅ CLI excelente para desarrollo local

🎯 Comparación: Consola vs SAM

Ejemplo: Crear Lambda en Consola (Manual)

  1. Navega a Lambda console
  2. Click "Create function"
  3. Ingresa nombre: {prefijo}-get-vinilos
  4. Selecciona runtime: Node.js 20.x
  5. Click "Create"
  6. Copia código en editor
  7. Click "Deploy"
  8. Ve a Configuration → Permissions
  9. Añade política de DynamoDB
  10. Ve a Configuration → Environment variables
  11. Añade TABLE_NAME=...
  12. Ve a API Gateway
  13. Create HTTP API
  14. Add integration con Lambda
  15. Configure route...

⏱️ Tiempo: 15-20 minutos (con errores, más)

Ejemplo: Crear Lambda con SAM (IaC)

GetVinilosFunction:
  Type: AWS::Serverless::Function
  Properties:
    FunctionName: !Sub '${StudentPrefix}-get-vinilos'
    CodeUri: src/handlers/get-vinilos/
    Handler: index.handler
    Runtime: nodejs20.x
    Environment:
      Variables:
        TABLE_NAME: !Ref VinilosTable
    Policies:
      - DynamoDBReadPolicy:
          TableName: !Ref VinilosTable
    Events:
      GetVinilos:
        Type: HttpApi
        Properties:
          Path: /vinilos
          Method: get

sam deploy

⏱️ Tiempo: 2-3 minutos

🔍 Ventajas Clave de SAM

1. Reproducibilidad

# Desplegar en dev
sam deploy --stack-name dev-vinilos --parameter-overrides Stage=dev

# Desplegar en prod (EXACTAMENTE igual)
sam deploy --stack-name prod-vinilos --parameter-overrides Stage=prod

2. Version Control

git log template.yaml

# Ver qué cambió
git diff HEAD~1 template.yaml

3. Rollback Automático

Si el deploy falla, SAM automáticamente revierte todos los cambios (transaccional).

4. Local Development

# Correr Lambda localmente (sin AWS)
sam local invoke GetVinilosFunction

# Correr API Gateway localmente
sam local start-api

5. Policy Templates

En lugar de escribir JSON de IAM:

# ❌ Manual: Escribir JSON complejo
Policies:
  - Version: '2012-10-17'
    Statement:
      - Effect: Allow
        Action:
          - dynamodb:GetItem
          - dynamodb:Scan
          - dynamodb:Query
        Resource: !GetAtt VinilosTable.Arn

# ✅ SAM: Usar template pre-definido
Policies:
  - DynamoDBReadPolicy:
      TableName: !Ref VinilosTable

📋 Recursos SAM Principales

1. AWS::Serverless::Function (Lambda)

Type: AWS::Serverless::Function
Properties:
  FunctionName: my-function
  Handler: index.handler
  Runtime: nodejs20.x
  CodeUri: src/
  Events:    # Triggers automáticos
    ApiEvent:
      Type: HttpApi

2. AWS::Serverless::HttpApi (API Gateway)

Type: AWS::Serverless::HttpApi
Properties:
  CorsConfiguration:
    AllowOrigins: ['*']
    AllowMethods: [GET, POST]

3. AWS::Serverless::SimpleTable (DynamoDB)

Type: AWS::Serverless::SimpleTable
Properties:
  PrimaryKey:
    Name: id
    Type: String

💡 Tip: Para recursos no-serverless (S3, RDS, etc.), usa tipos CloudFormation normales:

Type: AWS::S3::Bucket
Type: AWS::DynamoDB::Table

🛠️ Instalación de SAM CLI

Linux / macOS

# Opción 1: Homebrew (macOS)
brew tap aws/tap
brew install aws-sam-cli

# Opción 2: Script oficial (Linux/macOS)

🎓 Tu Primera Template SAM

Vamos a crear una Lambda básica:

AWSTemplateFormatVersion: '2010-09-09'
Transform: AWS::Serverless-2016-10-31
Description: Mi primera aplicación SAM

Resources:
  HelloWorldFunction:
    Type: AWS::Serverless::Function
    Properties:
      FunctionName: hello-world
      Handler: index.handler
      Runtime: nodejs20.x
      InlineCode: |
        exports.handler = async (event) => {
            return {
                statusCode: 200,
                body: JSON.stringify({ mensaje: '¡Hola desde SAM!' })
            };
        };

Outputs:
  FunctionName:
    Description: Lambda Function Name
    Value: !Ref HelloWorldFunction

Desplegar:

sam deploy --guided
