Ejercicio Opcional (Homework): S3 Trigger + Lambda + DynamoDB

Arquitectura Event-Driven para Procesamiento de Datos

🎯 Objetivo

Aprender arquitectura event-driven (orientada a eventos) creando un pipeline automático:
1. Subir un archivo CSV a S3
2. S3 dispara automáticamente una Lambda
3. Lambda procesa el CSV y guarda los datos en DynamoDB

Este ejercicio complementa los ejemplos en clase mostrando un patrón diferente: procesamiento asíncrono vs APIs síncronas.

📚 Conceptos Clave

Arquitectura Síncrona (Clase) vs Asíncrona (Homework)

Arquitectura Síncrona (API Gateway → Lambda)

Cliente HTTP → API Gateway → Lambda → DynamoDB
             ↓
        Respuesta inmediata
  • El cliente espera la respuesta
  • Lambda se ejecuta bajo demanda
  • Uso: APIs REST, aplicaciones web

Arquitectura Asíncrona (S3 Event → Lambda)

Usuario sube CSV → S3 Bucket
                    ↓ (Event Notification)
                  Lambda (se ejecuta automáticamente)
                    ↓
                  DynamoDB
  • El usuario no espera respuesta
  • Lambda se ejecuta automáticamente al detectar el evento
  • Uso: ETL, procesamiento de archivos, pipelines de datos

🆚 Comparación de Patrones

Característica API Gateway + Lambda S3 Trigger + Lambda
Invocación Síncrona (cliente espera) Asíncrona (automática)
Trigger Petición HTTP Evento S3 (upload)
Respuesta JSON inmediato No hay respuesta directa
Caso de uso APIs REST Procesamiento batch
Timeout Hasta 30s (API Gateway) Hasta 15 min (Lambda)
Ejemplo GET /vinilos Upload CSV → procesar

🛠️ Parte 1: Crear Bucket S3 para CSV

Paso 1: Crear Bucket

  1. Ve a S3 en la consola AWS
  2. Haz clic en "Create bucket"
  3. Bucket name: {tu-prefijo}-vinilos-csv
    - Ejemplo: juan-perez-vinilos-csv
  4. Region: eu-west-1 (Ireland)
  5. Block Public Access: ✅ Deja activado (este bucket es privado)
  6. Haz clic en "Create bucket"

✅ El bucket se crea instantáneamente.

🛠️ Parte 2: Crear Lambda de Procesamiento

Paso 1: Crear la Función Lambda

  1. Ve a Lambda en la consola AWS
  2. Haz clic en "Create function"
  3. Selecciona "Author from scratch"
  4. Function name: {tu-prefijo}-procesar-vinilos-csv
    - Ejemplo: juan-perez-procesar-vinilos-csv
  5. Runtime: Node.js 20.x
  6. Haz clic en "Create function"

Paso 2: Código de la Lambda

Reemplaza el código por defecto con este:

const { S3Client, GetObjectCommand } = require("@aws-sdk/client-s3");
const { DynamoDBClient } = require("@aws-sdk/client-dynamodb");
const { DynamoDBDocumentClient, PutCommand } = require("@aws-sdk/lib-dynamodb");

const s3Client = new S3Client({ region: "eu-west-1" });
const ddbClient = new DynamoDBClient({ region: "eu-west-1" });
const docClient = DynamoDBDocumentClient.from(ddbClient);

// ⚠️ IMPORTANTE: Reemplaza con tu prefijo
const TABLE_NAME = "TU-PREFIJO-vinilos";

exports.handler = async (event) => {
    console.log('Lambda invocada por S3 event');
    console.log('Event:', JSON.stringify(event, null, 2));

    try {
        // 1. Extraer información del evento S3
        const record = event.Records[0];
        const bucketName = record.s3.bucket.name;
        const objectKey = record.s3.object.key;

        console.log(`Procesando archivo: ${objectKey} del bucket: ${bucketName}`);

        // 2. Leer el archivo CSV desde S3
        const getObjectCommand = new GetObjectCommand({
            Bucket: bucketName,
            Key: objectKey
        });

        const s3Response = await s3Client.send(getObjectCommand);
        const csvContent = await streamToString(s3Response.Body);

        console.log('CSV content:', csvContent);

        // 3. Parsear CSV (formato: id,titulo,artista,precio,año)
        const lines = csvContent.trim().split('\n');
        const header = lines[0]; // Primera línea: encabezados
        const dataLines = lines.slice(1); // Resto: datos

        console.log(`CSV tiene ${dataLines.length} registros (sin contar encabezado)`);

        // 4. Procesar cada línea e insertar en DynamoDB
        let insertados = 0;
        let errores = 0;

        for (const line of dataLines) {
            if (!line.trim()) continue; // Saltar líneas vacías

            const [id, titulo, artista, precioStr, añoStr] = line.split(',').map(s => s.trim());

            // Validación básica
            if (!id || !titulo || !artista) {
                console.error(`Línea inválida (faltan campos): ${line}`);
                errores++;
                continue;
            }

            const vinilo = {
                id: id,
                titulo: titulo,
                artista: artista,
                precio: parseFloat(precioStr) || 0,
                año: parseInt(añoStr) || 0
            };

            try {
                const putCommand = new PutCommand({
                    TableName: TABLE_NAME,
                    Item: vinilo
                });

                await docClient.send(putCommand);
                console.log(`✅ Insertado: ${vinilo.titulo} (ID: ${vinilo.id})`);
                insertados++;

            } catch (error) {
                console.error(`❌ Error insertando ${vinilo.titulo}:`, error.message);
                errores++;
            }
        }

        // 5. Resultado final
        const resultado = {
            archivo: objectKey,
            totalLineas: dataLines.length,
            insertados: insertados,
            errores: errores
        };

        console.log('Procesamiento completado:', resultado);

        return {
            statusCode: 200,
            body: JSON.stringify(resultado)
        };

    } catch (error) {
        console.error('Error fatal:', error);

        return {
            statusCode: 500,
            body: JSON.stringify({
                error: error.message
            })
        };
    }
};

// Función auxiliar para convertir Stream a String
async function streamToString(stream) {
    return new Promise((resolve, reject) => {
        const chunks = [];
        stream.on('data', (chunk) => chunks.push(chunk));
        stream.on('error', reject);
        stream.on('end', () => resolve(Buffer.concat(chunks).toString('utf-8')));
    });
}

Paso 3: Configurar la Lambda

3.1 Actualizar el Timeout

Por defecto, Lambda tiene timeout de 3 segundos (muy poco para procesar archivos).

  1. Ve a la pestaña "Configuration"
  2. Haz clic en "General configuration"
  3. Haz clic en "Edit"
  4. Timeout: Cambia a 30 segundos
  5. Memory: Deja 128 MB (suficiente)
  6. Haz clic en "Save"

3.2 Añadir Permisos de DynamoDB

La Lambda necesita permiso para escribir en DynamoDB.

  1. Ve a la pestaña "Configuration" → "Permissions"
  2. Haz clic en el Role name (abre IAM en nueva pestaña)
  3. En IAM, haz clic en "Add permissions" → "Attach policies"
  4. Busca y selecciona: AmazonDynamoDBFullAccess
    - ⚠️ En producción usarías un policy más restrictivo (solo PutItem en tabla específica)
  5. Haz clic en "Add permissions"
  6. Cierra la pestaña de IAM

🛠️ Parte 3: Configurar S3 Event Notification

Ahora configuramos S3 para que dispare automáticamente la Lambda cuando se suba un archivo.

Paso 1: Añadir Permiso a Lambda (Resource Policy)

Primero, Lambda debe permitir que S3 la invoque.

  1. Ve a tu Lambda ({prefijo}-procesar-vinilos-csv)
  2. Pestaña "Configuration" → "Triggers"
  3. Haz clic en "Add trigger"
  4. Select a source: S3
  5. Bucket: Selecciona tu bucket ({prefijo}-vinilos-csv)
  6. Event type: All object create events (PUT, POST, COPY)
  7. ⚠️ Acknowledge: Marca la casilla:
    - "I acknowledge that using the same S3 bucket for both input and output..."
  8. Haz clic en "Add"

✅ Ahora S3 puede invocar tu Lambda automáticamente.

Paso 2: Verificar la Configuración en S3

  1. Ve a S3 → Tu bucket ({prefijo}-vinilos-csv)
  2. Pestaña "Properties"
  3. Desplázate hasta "Event notifications"
  4. Deberías ver una notificación creada con:
    - Event types: All object create events
    - Destination: Tu Lambda

✅ La configuración está completa.

🧪 Ejercicios Adicionales (Avanzado)

1. Validación Mejorada

Añade validación de datos:
- Precio debe ser > 0
- Año debe estar entre 1900 y año actual
- Titulo y artista no vacíos

2. Manejo de Duplicados

Modifica el código para:
- Detectar si un id ya existe en DynamoDB
- Solo insertar si es nuevo (o actualizar si existe)

Pista: Usa GetCommand antes de PutCommand:

const existing = await docClient.send(new GetCommand({
    TableName: TABLE_NAME,
    Key: { id: vinilo.id }
}));

if (existing.Item) {
    console.log(`ID ${vinilo.id} ya existe, saltando...`);
    continue;
}

3. Notificación de Resultados

Añade SNS (Simple Notification Service):
- Envía email al administrador cuando termine el procesamiento
- Incluye resumen: total procesado, insertados, errores

4. Carpetas en S3

Organiza archivos por fecha:
- uploads/2025-01-15/vinilos.csv
- Lambda detecta solo archivos en carpeta uploads/

Pista: Filtra por prefijo en Event Notification (S3).

5. Procesamiento de Archivos Grandes

Si el CSV tiene miles de líneas:
- Procesar por lotes (BatchWriteItem en DynamoDB)
- Añadir pausa entre lotes para no saturar DynamoDB

🐛 Troubleshooting

Lambda no se ejecuta al subir CSV

Causa 1: Event notification no configurado correctamente.

Solución:
- Ve a Lambda → Configuration → Triggers
- Verifica que aparece el bucket S3
- Ve a S3 → Bucket → Properties → Event notifications
- Verifica que el destino es tu Lambda

Causa 2: Lambda no tiene permiso (resource policy).

Solución:
- Borra el trigger en Lambda
- Vuelve a añadirlo (esto recrea el resource policy)

Error: "Access Denied" en logs

Causa: Lambda no tiene permisos para leer S3.

Solución:
1. Ve a Lambda → Configuration → Permissions → Role name
2. En IAM, añade policy: AmazonS3ReadOnlyAccess

Error: "Table does not exist"

Causa: Nombre de tabla incorrecto en el código.

Solución:
1. Ve a DynamoDB y verifica el nombre exacto de tu tabla
2. Actualiza TABLE_NAME en el código Lambda:
javascript const TABLE_NAME = "juan-perez-vinilos"; // EXACTO
3. Despliega el código de nuevo

Error: "ValidationException: Invalid attribute value type"

Causa: Tipo de dato incorrecto (ej: string en vez de number).

Solución:
- Usa parseFloat() para precios
- Usa parseInt() para años
- Verifica que CSV no tiene comillas extra

CSV se procesa pero no se insertan datos

Causa: Formato CSV incorrecto (líneas vacías, encoding).

Solución:
1. Abre el CSV en un editor de texto plano
2. Verifica:
- Primera línea: id,titulo,artista,precio,año
- Sin espacios antes/después de comas
- Sin líneas vacías entre registros
- Encoding UTF-8 (sin BOM)

Lambda se ejecuta dos veces

Causa: Event notification duplicado.

Solución:
- Ve a S3 → Bucket → Properties → Event notifications
- Borra notificaciones duplicadas
- Debe haber solo una
