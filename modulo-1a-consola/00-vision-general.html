Módulo 1 - Ejemplos en Consola AWS

Visión General del Caso de Uso Pedagógico

🎯 Objetivo

Construir paso a paso una API simple de catálogo de vinilos usando servicios AWS serverless, trabajando directamente en la consola de AWS para entender cómo funciona cada servicio antes de automatizar con Infrastructure as Code (SAM).

📚 Caso de Uso: "Catálogo de Vinilos"

Funcionalidad

Una API REST que devuelve un catálogo de vinilos con:
- Título del álbum
- Artista
- Precio
- Imagen de la portada

Ejemplo de respuesta

{
  "vinilos": [
    {
      "id": "1",
      "titulo": "Abbey Road",
      "artista": "The Beatles",
      "precio": 25.99,
      "imagen": "https://{prefijo}-vinilos-imagenes.s3.eu-west-1.amazonaws.com/abbey-road.jpg"
    },
    {
      "id": "2",
      "titulo": "Dark Side of the Moon",
      "artista": "Pink Floyd",
      "precio": 22.50,
      "imagen": "https://{prefijo}-vinilos-imagenes.s3.eu-west-1.amazonaws.com/dark-side.jpg"
    }
  ]
}

🏗️ Arquitectura Progresiva

Construiremos la arquitectura en 4 iteraciones, añadiendo un servicio AWS en cada paso:

Iteración 1: Lambda (hardcoded)
    ↓
Iteración 2: Lambda + DynamoDB
    ↓
Iteración 3: Lambda + DynamoDB + S3 (imágenes)
    ↓
Iteración 4: API Gateway + Lambda + DynamoDB + S3

Arquitectura Final

┌─────────────┐
│   Cliente   │
│ (curl/web)  │
└──────┬──────┘
       │ HTTP GET
       ↓
┌─────────────────┐
│  API Gateway    │
│ GET /vinilos    │
└────────┬────────┘
         │ invoke
         ↓
┌──────────────────┐      ┌─────────────────┐
│  Lambda          │─────→│   DynamoDB      │
│ get-vinilos      │ scan │ {prefijo}-      │
│                  │      │ vinilos         │
└──────────┬───────┘      └─────────────────┘
           │
           │ read URLs
           ↓
    ┌──────────────┐
    │      S3      │
    │  Imágenes    │
    │  (público)   │
    └──────────────┘

📋 Recursos AWS que Crearemos

Iteración Recurso Nombre Tipo
1 Lambda {prefijo}-get-vinilos Función Lambda (Node.js)
2 Tabla DynamoDB {prefijo}-vinilos NoSQL Table
3 Bucket S3 {prefijo}-vinilos-imagenes Object Storage
4 API Gateway {prefijo}-vinilos-api HTTP API

Nota: {prefijo} es tu identificador personal (ej: juan-perez)

⏱️ Tiempo Estimado

Iteración Duración Acumulado
Iteración 1: Lambda hardcoded 15 min 15 min
Iteración 2: + DynamoDB 20 min 35 min
Iteración 3: + S3 imágenes 20 min 55 min
Iteración 4: + API Gateway 15 min 70 min

Total: ~1 hora 10 minutos (incluyendo explicaciones del instructor)

🎓 Aprendizajes por Iteración

Iteración 1: Lambda Básica

  • ✅ Qué es AWS Lambda
  • ✅ Cómo crear una función desde consola
  • ✅ Cómo probar una función Lambda
  • ✅ Estructura de eventos y respuestas
  • ✅ Logs en CloudWatch

Iteración 2: Lambda + DynamoDB

  • ✅ Qué es DynamoDB (NoSQL)
  • ✅ Crear tablas y añadir items
  • ✅ AWS SDK para Node.js (v3)
  • ✅ IAM Roles y políticas de permisos
  • ✅ Operación Scan en DynamoDB

Iteración 3: Lambda + DynamoDB + S3

  • ✅ Qué es S3 y para qué se usa
  • ✅ Crear buckets y subir objetos
  • ✅ Bucket policies (acceso público)
  • ✅ URLs de objetos S3
  • ✅ Combinar datos de múltiples fuentes

Iteración 4: API Gateway

  • ✅ Qué es API Gateway
  • ✅ HTTP APIs vs REST APIs
  • ✅ Crear rutas y métodos
  • ✅ Integración con Lambda
  • ✅ CORS (Cross-Origin Resource Sharing)
  • ✅ Probar APIs con curl/Postman

🚀 Siguientes Pasos

Una vez completadas las 4 iteraciones en consola, migraremos todo a AWS SAM (Serverless Application Model) para automatizar el despliegue con Infrastructure as Code.

📝 Nomenclatura Importante

SIEMPRE usa tu prefijo personal al inicio del nombre de cada recurso:
- ✅ juan-perez-get-vinilos
- ✅ juan-perez-vinilos
- ✅ juan-perez-vinilos-imagenes
- ❌ get-vinilos-juan-perez (prefijo al final)
- ❌ vinilos (sin prefijo, colisión con otros alumnos)

Esto evita colisiones entre estudiantes que trabajan en la misma cuenta AWS.
